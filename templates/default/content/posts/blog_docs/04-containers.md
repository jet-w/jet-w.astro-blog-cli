# 容器语法

容器是本博客的特色功能，用于创建醒目的提示框、警告框等，让文章更加清晰易读。

## 基本语法

```markdown
::: 类型 可选标题
容器内容
:::
```

## 容器类型

本博客支持 6 种容器类型：

### tip - 提示

```markdown
::: tip
这是一个提示信息。
:::

::: tip 自定义标题
这是带自定义标题的提示。
:::
```

::: tip
这是一个提示信息。
:::

::: tip 自定义标题
这是带自定义标题的提示。
:::

### info - 信息

```markdown
::: info
这是一条信息说明。
:::

::: info 补充说明
关于这个功能的更多信息...
:::
```

::: info
这是一条信息说明。
:::

::: info 补充说明
关于这个功能的更多信息...
:::

### note - 注意

```markdown
::: note
请注意这一点。
:::

::: note 注意事项
使用时需要注意以下几点...
:::
```

::: note
请注意这一点。
:::

::: note 注意事项
使用时需要注意以下几点...
:::

### warning - 警告

```markdown
::: warning
这是一条警告信息。
:::

::: warning 版本兼容性
此功能仅在 v2.0 以上版本可用。
:::
```

::: warning
这是一条警告信息。
:::

```markdown
::: danger
危险操作，请谨慎！
:::

::: danger 数据丢失风险
此操作将永久删除数据，无法恢复！
:::
```


:::

## 容器内容

### 支持多行文本

```markdown
::: tip 多行内容
第一行内容。

第二行内容。

第三行内容。
:::
```

::: tip 多行内容
第一行内容。

第二行内容。

第三行内容。
:::

### 支持列表

```markdown
::: info 功能特点
- 特点一：快速响应
- 特点二：简单易用
- 特点三：功能强大
:::
```

::: info 功能特点
- 特点一：快速响应
- 特点二：简单易用
- 特点三：功能强大
:::

### 支持代码

```markdown
::: tip 代码示例
使用以下代码：
\`\`\`javascript
console.log('Hello');
\`\`\`
:::
```

::: tip 代码示例
使用以下代码：
```javascript
console.log('Hello');
```
:::

### 支持链接和格式

```markdown
::: note 参考资料
更多信息请查看 [官方文档](https://example.com)。

重点内容用 **粗体** 标记。
:::
```

::: note 参考资料
更多信息请查看 [官方文档](https://example.com)。

:::

### 补充说明

::: details 实现原理
这个功能基于以下技术实现：

1. 使用 remark-directive 解析自定义语法
2. 通过 remark-containers 插件转换为 HTML
3. 使用 Tailwind CSS 进行样式化

## 样式对照表

| 类型 | 颜色 | 图标 | 适用场景 |
|:---|:---|:---|:---|
| tip | 蓝色 | 💡 | 有用的建议、技巧 |
| info | 蓝色 | ℹ️ | 补充信息、说明 |
| note | 灰色 | 📝 | 注意事项、备注 |
| warning | 黄色 | ⚠️ | 警告、注意 |
| danger | 红色 | 🚨 | 危险操作、严重警告 |
| details | 灰色 | 📋 | 可折叠的详细内容 |

## 最佳实践

### 1. 选择合适的类型

```markdown
<!-- 好：选择合适的类型 -->
::: tip 性能提示
启用缓存可以提升 50% 的加载速度。
:::

<!-- 不好：类型与内容不匹配 -->
::: danger 性能提示
启用缓存可以提升 50% 的加载速度。
:::
```

### 2. 保持内容简洁

```markdown
<!-- 好：简洁明了 -->
::: warning
Node.js 版本需要 >= 18.0
:::

<!-- 不好：内容过长 -->
::: warning
这是一个非常长的警告信息，包含了很多不必要的内容...
（应该拆分或简化）
:::
```

### 3. 添加有意义的标题

```markdown
<!-- 好：标题说明内容 -->
::: info 环境要求
- Node.js >= 18.0
- npm >= 9.0
:::

<!-- 一般：使用默认标题 -->
::: info
- Node.js >= 18.0
- npm >= 9.0
:::
```

### 4. 避免过度使用

一篇文章中不要使用过多的容器，否则会分散读者注意力。建议：

- 每个章节最多 2-3 个容器
- 只在确实需要强调的地方使用
- details 容器适合放置参考信息
